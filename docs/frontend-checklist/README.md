# Checklist de datos reales para aterrizar el frontend

Este documento está pensado para que puedas reunir, completar y pasar toda la información real que necesitamos para empezar a sustituir placeholders, estilos temporales y contenido genérico en el frontend.

La lógica es simple:

- el **README principal del proyecto** sigue documentando el repositorio;
- este **README secundario** se usa como checklist de handoff para la identidad visual, el contenido y los datos reales del sitio.

> Ya tienes colores y tipografía. Perfecto: eso resuelve una parte importante del sistema visual. Con esta checklist podemos completar el resto del frontend con criterio.

---

## Cómo usar este documento

- Marca cada elemento con `[x]` cuando ya lo tengas definido.
- Añade enlaces, archivos, notas o ejemplos cuando aplique.
- Si algo todavía no existe, puedes dejar una decisión provisional.
- Si un dato solo aplica a la versión en español o a la inglesa, indícalo.

## 1. Identidad visual

### Marca base
- [ ] Nombre oficial de la marca.
- [ ] Variante corta del nombre para navbar, footer o favicon.
- [ ] Tagline o descriptor principal.
- [ ] Propuesta de valor en una frase.
- [ ] Tono de marca en 3 a 5 adjetivos.
- [ ] Palabras que sí representan la marca.
- [ ] Palabras o enfoques que no quieres usar.

### Sistema visual
- [x] Colores reales de identidad visual.
- [x] Tipografía real.
- [ ] Escala tipográfica deseada para H1, H2, H3, párrafos y botones.
- [ ] Pesos tipográficos permitidos.
- [ ] Estilo de botones.
- [ ] Estilo de cards o contenedores.
- [ ] Radio de bordes.
- [ ] Sombras, brillos o decisión de no usarlos.
- [ ] Sistema de espaciado o referencias de composición.
- [ ] Reglas de iconografía.
- [ ] Reglas de fotografía o ilustración.
- [ ] Nivel de animación o motion deseado.

### Archivos y assets
- [ ] Logo principal en SVG o vector.
- [ ] Versiones alternativas del logo.
- [ ] Favicon.
- [ ] Variantes para fondo claro y fondo oscuro.
- [ ] Carpeta o enlace con assets de marca.

## 2. Dirección de diseño

- [ ] 3 a 5 referencias visuales de webs que te gusten.
- [ ] Qué te gusta de cada referencia.
- [ ] Qué no quieres replicar.
- [ ] Sensación buscada: premium, sobria, creativa, tecnológica, estratégica, cercana, etc.
- [ ] Qué debe tener más peso visual: agencia, marca personal, desarrollo web o mezcla concreta.
- [ ] Preferencias concretas para móvil.
- [ ] Preferencias concretas para escritorio.

## 3. Estructura real de la marca o negocio

Ahora mismo el sitio sugiere una marca híbrida entre agencia y perfil profesional. Para ajustar bien el frontend, conviene concretar esto.

- [ ] La marca se presenta como una sola entidad o como dos líneas de servicio.
- [ ] Qué debe entender un visitante en los primeros 5 segundos.
- [ ] Servicio principal que quieres vender ahora.
- [ ] Servicio secundario o complementario.
- [ ] Público objetivo principal.
- [ ] Público objetivo secundario.
- [ ] Alcance geográfico real.
- [ ] Idioma principal del negocio.

## Recomendación práctica

El mejor siguiente paso es que me compartas primero este paquete mínimo:

1. colores;
2. tipografías;
3. logo;
4. H1 y subtítulo reales de la home;
5. menú definitivo;
6. servicios reales;
7. 2 o 3 proyectos reales;
8. email, redes y CTA principal;
9. 2 o 3 referencias visuales;
10. una explicación breve de si debe pesar más la **agencia**, la **marca personal**, el **desarrollo web** o una mezcla concreta.

Con eso ya se puede empezar una primera ronda realista de ajustes visuales y estructurales en el frontend.
